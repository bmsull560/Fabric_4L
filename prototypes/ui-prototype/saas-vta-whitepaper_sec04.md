## 4. Digital Preference Elicitation Engine

The additive value model sits at the heart of Value Tree Analysis. For a decision alternative $a$ with consequence vector $(x_1(a), \ldots, x_n(a))$, the total value is

$$V(a) = \sum_{i=1}^{n} w_i \, v_i\bigl(x_i(a)\bigr)$$

where $w_i$ is the weight of attribute $X_i$ and $v_i$ is the single-attribute value function normalised to $[0,1]$ [^1^]. Eliciting these two parameter families — value functions and weights — has historically required a trained decision analyst to conduct structured interviews, often consuming days of calendar time and introducing substantial cost and friction. This chapter maps every manual elicitation method from the classical VTA repertoire to its interactive digital equivalent, demonstrating that the interview room can be replaced by a self-service web interface without sacrificing methodological rigour.

The transformation rests on a simple but powerful principle: each manual technique asks the decision maker (DM) to produce a numerical judgment relative to a reference point. A direct rating assigns a score between worst and best; SMART assigns points relative to the least important attribute; AHP assigns ratios comparing pairs. Every one of these operations has a natural interface primitive — slider, drag-and-drop card, pairwise toggle — that can be rendered in a browser with real-time feedback [^2^]. The result is not a simplification of the methodology but a change in delivery modality: from analyst-prompted to interface-guided, from batch calculation to reactive computation, from single-user to collaborative.

### 4.1 Value Function Elicitation

The purpose of value function elicitation is to map attribute performance levels to desirability scores. The 2002 VTA document describes five core techniques; each translates to a distinct digital interaction pattern.

**Direct Rating: the visual slider interface.** In the manual method, the DM ranks alternatives on a single attribute, identifies the worst and best, assigns them 0 and 100 respectively, and places the remaining alternatives proportionally [^1^]. The digital equivalent is a continuous slider rendered on a 0–100 scale with anchored endpoints. Modern survey platforms have refined this pattern extensively: Qualtrics offers ten built-in graphic slider types including gauge, thermometer, and traffic-light metaphors [^2^]; SurveySparrow implements thermometer, gauge meter, and smiley variants that transform abstract numerical rating into visually concrete interactions [^3^]. In a VTA SaaS context, the slider is contextualised to the attribute being assessed — a "salary" slider might anchor at the lowest and highest offer values, while a "working conditions" slider uses emoji endpoints (😟 to 😀). As the DM adjusts each slider, a real-time bar chart updates to show the relative positioning of all alternatives on that attribute, providing immediate visual confirmation that spacing reflects preference strength [^4^].

**Category Estimation: dropdown and emoji selectors.** Where direct rating asks for a position on a continuous scale, category estimation collapses the response space to a finite set of labelled levels [^1^]. The digital implementation replaces verbal categories with visually distinct selectors — a row of emoji-grade chips (e.g., poor → fair → good → excellent), a segmented control, or a colour-graded button group. Each category maps to a normalised value score in the backend. The FITradeoff method demonstrates that even partial categorical information — knowing only that option A is preferred to option B without assigning exact values — can be sufficient to eliminate dominated alternatives through linear programming, reducing the number of judgments required from the DM [^5^].

**Graphical Curve Drawing: the D3.js value function editor.** For attributes with continuous numerical scales (price, distance, time), the DM may wish to specify a non-linear value function — for instance, diminishing marginal value for salary above a threshold. The manual approach required drawing a curve on paper; the digital equivalent is an interactive piecewise-linear editor powered by D3.js or a similar vector graphics library [^6^]. M-MACBETH provides the reference implementation: users click and drag control points on a two-axis graph (performance level on the horizontal axis, partial value score on the vertical), with automatic consistency checking against the DM's qualitative judgments [^7^]. An online MACBETH tool extends this with least-squares curve fitting, automatically classifying the function shape as concave, convex, or linear [^8^]. The resulting piecewise-linear function is transparently displayed alongside its mathematical definition — each segment's slope and intercept listed for audit and reproducibility [^9^].

**Difference Standard Sequence and Bisection: guided midpoint-finding workflows.** These indifference methods construct a value scale by finding attribute levels that are equally spaced in preference strength [^1^]. In the difference standard sequence, the DM identifies a chain of levels $x_0, x_1, \ldots, x_n$ such that each step represents an equal increment in value. In bisection, the DM recursively finds the midpoint between two levels in value space. Digitally, these become guided workflows: the system presents two anchored endpoints, asks the DM to position a midpoint marker, validates the response against consistency constraints, and iterates. Binary-search-style visual feedback — a splitting line that the DM drags until indifference is declared — replaces the abstract questioning of the analyst-led session. Web-HIPRE demonstrated the feasibility of such browser-based indifference elicitation as early as the 2000s [^10^].

**AI-assisted value inference: preference learning from partial inputs.** The final method is not from the 2002 document but represents a capability unique to the digital paradigm. Variational preference learning (VPL), developed at the University of Washington, predicts a user's preferences as they interact with the system and progressively refines value function estimates [^11^]. Where the DM has rated three alternatives on an attribute, a preference learning backend can infer the likely ratings for the remainder, presenting them as draft values that the DM may accept or override. Research on current preference learning algorithms (DPO, RLHF) reveals they achieve less than 60% ranking accuracy on common datasets, with "alignment gaps" of up to 59 percentage points [^12^]. This means AI inference should serve as a suggestion engine, not a replacement for explicit judgment — the structured elicitation interface remains the primary input channel, with machine learning reducing the number of required inputs rather than eliminating human judgment.

The breadth of this mapping is significant: every technique described in Section 4.1 of the source document has a production-ready digital equivalent. The table reveals a pattern — methods that were cognitively demanding in the manual setting (curve drawing, bisection) become more tractable when guided by visual feedback and automated consistency checking. The slider, the drag-and-drop card, and the interactive graph are the three fundamental interaction primitives that replace the analyst's prompting voice.

### 4.2 Weight Elicitation Methods

Weights in the additive model describe the relative importance of attributes. The 2002 VTA document describes four families of weight elicitation: SMART, rank-based methods, SWING, and AHP. Each maps to a distinct digital workflow.

**SMART: the interactive 10-point least-important scale.** The Simple Multi-Attribute Rating Technique asks the DM to assign 10 points to the least important attribute, then rate all other attributes proportionally relative to that baseline [^1^]. The weight for attribute $i$ is computed as:

$$w_i = \frac{p_i}{\sum_{j=1}^{n} p_j}$$

where $p_i$ is the point total assigned to attribute $i$ and $p_{\min} = 10$. The digital interface presents this as a point-allocation panel: a column of numeric input fields (or stepper controls) pre-populated with 10 for the least important attribute, with real-time normalisation displaying the resulting weight percentages. A visual bar chart beside each attribute grows or shrinks as points are adjusted, ensuring the DM sees the weight consequence of every rating change. SMART has been implemented in mobile decision-support applications with all calculations performed in the background, confirming the algorithmic simplicity of the approach [^14^].

**SWING: visual worst-to-best trade-off simulation.** The SWING method begins from the worst hypothetical alternative — all attributes at their minimum level — and asks the DM to choose which single attribute to "swing" to its best level [^1^]. This attribute receives 100 points; subsequent attributes are rated proportionally. The method requires the DM to consider the full range of variation for each attribute, making it more context-sensitive than simple ranking. The digital implementation uses animated vignette cards: the screen displays a baseline card showing all attributes at worst levels, and the DM clicks the attribute to improve first. The selected attribute animates to its best level while others remain fixed, creating a vivid sense of incremental gain. The DM then clicks the next attribute to improve, and a relative-importance slider appears for scoring [^15^]. Researchers implementing online SWING elicitation found that draggable vignettes for ranking combined with slider-based proportional rating significantly improved usability over text-entry forms [^16^].

**AHP: pairwise comparison matrix with real-time consistency guidance.** The Analytic Hierarchy Process uses ratio-scale pairwise comparisons. For $n$ attributes, the DM produces a comparison matrix $A$ where element $a_{ij}$ expresses the ratio of attribute $i$'s importance to attribute $j$'s importance on Saaty's 1–9 scale [^1^]. Weights are derived as the normalised principal eigenvector of $A$:

$$A \mathbf{w} = \lambda_{\max} \mathbf{w}$$

Consistency is measured by the Consistency Ratio:

$$\text{CR} = \frac{\text{CI}}{\text{RI}} = \frac{(\lambda_{\max} - n)/(n - 1)}{\text{RI}}$$

where RI is the random index for matrix size $n$. Saaty's threshold of CR $\leq$ 0.1 defines acceptable consistency [^17^]. The digital AHP interface renders this as a grid of pairwise comparison cells, each containing a slider that the DM moves to indicate relative importance. As each judgment is entered, three real-time feedback mechanisms activate: (1) the CR is recalculated and displayed in colour-coded form (green $\leq$ 0.1, yellow 0.1–0.2, red $>$ 0.2); (2) the three most inconsistent judgments are highlighted with recommended revision values; (3) the derived weight vector updates a bar chart in a side panel [^18^]. AHP-OS, a free web-based system serving over 5,000 registered users, implements all of these capabilities including group decision support with Shannon entropy-based consensus measurement [^19^]. Research on a 2D plot UI widget for AHP showed that transitivity-enforcing visual interfaces reduced required clicks by 36% compared to traditional questionnaire widgets while maintaining comparable consistency ratios [^20^]. In a clinical study of 123 participants comparing five preference elicitation methods, the adaptive pairwise approach (PAPRIKA) received the highest ratings for clarity, usability, and perceived ability to express preferences [^21^].

**Rank-based methods: drag-to-rank with automatic weight computation.** When the cognitive burden of numerical estimation is excessive, rank-based methods require only an ordering of attributes and compute weights through closed-form formulas [^1^]:

| Method | Unnormalised Weight | Formula |
|---|---|---|
| Rank Sum | $w_j = n - R_j + 1$ | Linear decay with rank position |
| Reciprocal Rank | $w_j = 1 / R_j$ | Inverse proportion to rank |
| Rank Exponent | $w_j = (n - R_j + 1)^p$ | Parameter $p$ controls distribution steepness |

The digital interface presents attributes as draggable cards. The DM reorders them by importance; weights compute instantly. A parameter slider for the exponent $p$ in the rank-exponent method lets the DM adjust weight distribution steepness interactively, with the bar chart updating in real time. SMARTER (SMART Exploiting Ranks) — a variant that requires only rank ordering and automatically applies rank-order-centroid weights — achieves approximately 98% of SMART's accuracy without requiring any difficult numerical judgments from the DM [^22^], making it ideal for large-scale stakeholder surveys where individual attention is limited.

**Hierarchical weight management: local and global weight views.** In a multi-level value tree, weights can be defined locally (within each branch) or globally (across all leaf attributes). Local weights are elicited at each node; global weights are computed by multiplying local weights down the tree. The digital interface must present both views: a split-panel showing the value tree on the left and the weight editor on the right. When the DM selects a parent node, the editor displays its children's local weights; when a leaf node is selected, the global weight (computed from the product of all ancestor local weights) is displayed with a full derivation trail. Automatic roll-up calculations ensure that any change to a local weight propagates to all dependent global weights and refreshes the overall alternative rankings.

### 4.3 Imprecise Preference Handling

The methods above assume the DM can state precise numerical values. In practice, decision makers are often uncertain about their preferences, and group decisions inherently involve conflicting views that no single point estimate can capture. The 2002 VTA document introduces the PRIME method (Preference Ratios in Multiattribute Evaluation) as a structured approach to handling imprecision [^1^]. Translated into SaaS capabilities, PRIME's interval-based approach becomes the platform's most distinctive enterprise feature — the capability that differentiates a purpose-built VTA tool from generic survey or BI platforms.

**Interval-based preference input: range sliders instead of point estimates.** Rather than asking a DM to specify that attribute A is "exactly 3 times more important than attribute B," the digital interface presents a range slider allowing the judgment "between 2 and 4 times." Each interval-bound judgment creates a linear constraint on the weight space. The system solves these constraints using linear programming to compute the feasible region. The Deck of Cards Method (DCM) digital implementation — available in HELDA software — demonstrates a related pattern: users insert "white cards" between criteria to express interval-based preference strength [^24^]. The CROC (Cardinal and Rank Ordering of Criteria) method uses graphical sliders with "clouded regions" representing interval uncertainty [^25^]. Power BI's Numeric Range Parameters provide a direct precedent: slicer-based range inputs allow users to explore scenarios with adjustable confidence intervals [^26^].

**PRIME-style ordinal and ratio statements: structured preference language.** PRIME supports three types of imprecise statements [^1^]: (1) ordinal rankings ($v(x_j) \geq v(x_k)$), (2) ratio-of-value-difference intervals ($L \leq \frac{v(x_j) - v(x_k)}{v(x_l) - v(x_m)} \leq U$), and (3) holistic comparisons. The digital interface renders these as a structured preference language: ordinal statements are created by drag-and-drop ranking; ratio statements by interval sliders comparing two pairs of alternatives; holistic statements by overall preference toggles between complete alternative profiles. Each statement adds a constraint to the linear programme. The system continuously reports how many constraints have been entered and the dimensionality of the remaining feasible space, giving the DM a sense of progress toward a determinate ranking.

**Value intervals for alternatives: linear programming computation.** From imprecise inputs, the system computes best-case and worst-case total values for each alternative by solving:

$$V_j^{\max} = \max_{\mathbf{w} \in \mathcal{W}} \sum_{i=1}^{n} w_i \, v_i(x_{ji}), \qquad V_j^{\min} = \min_{\mathbf{w} \in \mathcal{W}} \sum_{i=1}^{n} w_i \, v_i(x_{ji})$$

where $\mathcal{W}$ is the feasible weight region defined by all elicited constraints. The result is an interval $[V_j^{\min}, V_j^{\max}]$ for each alternative. These intervals are displayed as horizontal error-bar segments on the ranking chart, immediately communicating which alternatives are clearly separable and which overlap in value.

**Dominance analysis: absolute and pairwise dominance detection.** Two dominance concepts operate on value intervals [^1^]. *Absolute dominance* holds when one alternative's entire value interval lies above another's:

$$V_j^{\min} > V_k^{\max} \;\Rightarrow\; \text{alternative } j \text{ absolutely dominates alternative } k$$

*Pairwise dominance* is a weaker condition: alternative $j$ pairwise-dominates $k$ if, for every feasible weight vector in $\mathcal{W}$, $j$'s total value exceeds $k$'s. The digital interface automatically computes both dominance structures and renders them as an interactive matrix: dominated alternatives are greyed out, absolutely dominated rows show a clear "eliminated" badge, and hovering over a cell reveals the linear programming certificate proving the dominance relationship. DecideIT — the closest existing implementation — provides embedded sensitivity analysis through contraction levels that zoom into overlapping intervals to identify where dominance can be established [^27^].

**Decision rules under uncertainty: maximax, maximin, minimax regret, central value.** When intervals overlap and no dominance relationship holds, the platform offers four decision rules for breaking the tie [^1^]: *maximax* selects the alternative with the largest possible upper bound (optimistic); *maximin* selects the one with the largest lower bound (pessimistic); *minimax regret* selects the one minimising the maximum possible regret across all feasible weights; *central value* selects the one with the highest midpoint of its value interval. The digital interface presents these as a selectable rule set with animated visual explanations — for example, maximax highlights the right endpoint of each interval bar, maximin highlights the left, and minimax regret shows a "regret gap" animation illustrating the worst-case loss for each alternative.

**Diagram Description: Digital Preference Elicitation Engine Architecture.** The Digital Preference Elicitation Engine is structured as a three-layer system. At the top, the **Interaction Layer** presents five method-specific interfaces — sliders for direct rating, emoji selectors for category estimation, D3.js curve editors for value function drawing, vignette cards for SWING, and pairwise comparison matrices for AHP. Each interface feeds into the **Computation Layer**, which runs three parallel solver types: an eigenvector engine for AHP weights, a linear programming solver for PRIME interval computations, and a preference-learning inference module for AI-assisted draft values. The Computation Layer outputs to the **Visualisation Layer**, which renders real-time weight bars, value interval charts with error bands, dominance matrices with highlighting, and decision-rule comparisons with animated transitions. All three layers operate reactively: every user input propagates through computation to visual feedback within sub-second latency, creating the "living model" experience that distinguishes digital VTA from its batch-process predecessor.

The impossibility of stating exact preferences is not a failure mode — it is the normal condition of serious decision-making. PRIME's interval framework, implemented as a cloud-native service with real-time dominance computation and collaborative interval aggregation, addresses this condition directly. Where conventional tools force premature precision, a PRIME-enabled VTA SaaS preserves epistemic honesty. The DecideIT software demonstrates that the underlying mathematics — linear programming over feasible weight regions, contraction analysis for sensitivity, second-order belief distributions for discrimination — are computationally tractable [^31^]. What has been missing is the packaging: a browser-based, multi-user, visually intuitive platform that makes imprecise preference handling accessible to non-specialists. That gap is the strategic opportunity.
