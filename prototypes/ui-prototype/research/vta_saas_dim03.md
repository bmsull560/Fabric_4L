## Finding 23: MCDA Performance Matrices as Fundamental Decision Tools

Claim: In MCDA, the performance matrix is "the most basic alternative approach" where each alternative is a row and each criterion is a column. Performance scores are usually normalized (0-100), with scores implemented using continuous scales or discrete levels (low/medium/high). The weighted-sum model (linear additive equation) is the most common aggregation approach, combining criterion weights with alternative scores. [^110^]

In the table/matrix, each alternative is a row and each criterion is a column... # Summary: Digital Translation of VTA Attribute Specification & Performance Matrix

## 1. Core SaaS Capabilities for Performance Matrices

Modern SaaS platforms translate the Value Tree Analysis performance matrix into digital workflows through several key capabilities:

| VTA Concept | SaaS Implementation |
|------------|-------------------|

| Natural attributes | Direct data import from integrations, automated metrics, KPI feeds |
| Constructed attributes | Rubric-based scales, subjective scoring with descriptor anchors, pairwise comparisons |
| Proxy attributes | Indirect indicators, surrogate metrics, AI-extracted features |

## 2. Weighted Scoring as the Dominant Aggregation Model

The weighted-sum model (also called points systems, linear additive models, or scoring rubrics) is the most prevalent approach across all evaluated platforms [^110^] [^42^] [^45^]. Key properties include:

- **Weight normalization**: Criterion weights sum to 100% (or 1.0)
- **Score normalization**: Performance typically scored on 0-100 or 1-5 scales
- **Aggregation**: Linear combination: Total Score = sum(weight_i * score_i) for all criteria i
- **Variations**: Section-level weighting, question-level weighting, multi-scorer averaging
