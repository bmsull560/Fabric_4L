# Journey Launch Release Checklist

**Journey:** account → signals → evidence → driver → calculator → business case  
**Updated:** 2026-05-07

## Launch SLOs

| SLO | Target | Owner |
|---|---:|---|
| Success rate | >= 99% over 15m | Frontend Platform |
| p95 latency | <= 12s over 15m | Frontend Platform + Layer4 |
| Non-empty response ratio | 100% over 15m | AI Experience |

## CI / Release gates

- `pnpm run test:e2e:journey-launch` must pass.
- `pnpm run test:journey-slo-gate` must pass using `JOURNEY_SLO_REPORT_PATH` generated by synthetic monitor runs.
- Prometheus alerts `JourneySuccessRateSLOViolation`, `JourneyP95LatencySLOViolation`, and `JourneyNonEmptyResponseSLOViolation` must be green.

## Deterministic synthetic data

- Use `apps/web/e2e/fixtures/synthetic-tenant-data.ts`.
- Seed with fixed tenant/account IDs (`tenant-synthetic-ci`, `acct-synthetic-alpha`).
- Do not mutate shared fixture values in test bodies; add new fixture versions instead.

## Rollback criteria

Rollback is required when any of the following occurs in production:

1. Success rate < 99% for 10+ minutes.
2. p95 latency > 12s for 10+ minutes.
3. Any empty required response payload during the clickpath.
4. Export-ready business case can be generated without valid evidence chain.

## Rollback owner path

1. **Incident commander:** On-call SRE.
2. **Feature rollback:** Frontend Platform owner disables feature flag / route entry.
3. **Data rollback:** Layer4 owner reverts journey payload contract change.
4. **Sign-off to re-enable:** Product + Engineering manager + SRE.

## Intelligence Workspace deferred tab rollout flags

The following tabs are intentionally excluded from production navigation until the owning teams complete full production states (loading, empty, error, success) and acceptance:

| Tab ID | Rollout flag | Default in production | Owner |
|---|---|---|---|
| `ontology-match` | `VITE_ENABLE_IW_ONTOLOGY_MATCH_TAB` | `false` | Intelligence Workspace / Layer 4 Agents |
| `alternatives` | `VITE_ENABLE_IW_ALTERNATIVES_TAB` | `false` | Intelligence Workspace / Product Strategy |
| `solution-cost` | `VITE_ENABLE_IW_SOLUTION_COST_TAB` | `false` | Intelligence Workspace / Value Modeling |

Release managers must keep all three flags disabled in production until explicit sign-off from the listed owner.
