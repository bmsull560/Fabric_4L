# Production-Readiness Policy Enforcement Matrix

This matrix inventories every file in `config/production-readiness/` and maps it to ownership, CI enforcement surface, and failure semantics.

| Policy file | Owning module / test | CI job that enforces it | Failure semantics |
|---|---|---|---|
| `config/production-readiness/billing_metering_policy.json` | `scripts/ci/validate_production_readiness_plan.py` (`billing_metering` gate), billing evidence patterns (`*billing*.py`, `*billing*.md`, `*billing*.yml`) | `.github/workflows/prod-readiness.yml` â†’ `setup` job (`make gates-validate-policy`) and release validation (`make gate-release-policy` consumes generated evidence) | Missing/invalid JSON, empty `productionEvidenceRequired`, secret-like values, or no billing evidence match causes validator exit `1`, failing workflow. |

| `config/production-readiness/feature_flag_rollout_policy.json` | `scripts/ci/validate_production_readiness_plan.py` (`feature_flags` and `sdk_cli` gates), `services/layer4-agents/tests/test_feature_flags.py` | `.github/workflows/prod-readiness.yml` and test workflows executing Layer 4 / release tests | Validator fail-closed on missing policy/docs/evidence; CI fails immediately on non-zero validator exit. |
