# Subprocessor Register Template

Use this template to maintain an auditable list of subprocessors.

## Register Fields

| Field | Description | Required |
|---|---|---|
| Subprocessor name | Legal name of vendor | Yes |
| Service provided | Functional purpose (e.g., hosting, analytics, support) | Yes |
| Data categories | Types of customer data processed | Yes |
| Processing purpose | Why data is processed | Yes |
| Data subjects | Customer end users, admins, employees, etc. | Yes |
| Hosting regions | Countries/regions where data is stored/processed | Yes |
| Transfer mechanism | SCCs, adequacy decision, or equivalent | If cross-border |
| Security attestations | SOC 2, ISO 27001, etc. | Preferred |
| Contract start / renewal date | Commercial lifecycle dates | Yes |
| DPA executed | Y/N + date | Yes |
| Status | Active, pending, retired | Yes |
| Last review date | Most recent review completion date | Yes |
| Owner | Internal accountable owner | Yes |

## Publication Guidance

- Maintain an internal master register with complete details.
- Publish a customer-facing register or notice page that is updated when subprocessors change.
- Send advance notice for material subprocessor additions when contractually required.
