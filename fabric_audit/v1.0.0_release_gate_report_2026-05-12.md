## 5. Root Cause Analysis

### Blocker 1: Branch Protection Misconfiguration (P1)

`mandatory-security-regression` is not configured as a required status check in GitHub branch protection. The `branch-protection-validation.yml` workflow documents what checks should be required, but the actual branch protection settings only list `ship-decision`.

### Blocker 2: CI Test Collection Failure (P0)

`services/layer2-extraction/tests/conftest.py:90` raises a `RuntimeError` during pytest collection, preventing any backend tests from running. The error states:

> "Layer 2 tests require the real value_fabric.shared package. Run pytest from the repository root or install tests/requirements-test.txt; do not stub value_fabric because it breaks repository-level collection."

This suggests the CI environment is not installing the package in editable mode, or the Python path is not set correctly for repository-level test discovery.

### Blocker 3: Security Regression Gate Database Configuration (P0)

`app/tests/test_auth_enforcement.py` tests fail with `ProductionPersistenceNotConfigured: database_url must be configured when mock_persistence is false.` The CI workflow for mandatory security regression does not configure a test database URL, causing 10 of 54 auth enforcement tests to fail.

### Blocker 4: Merge Conflicts in 9 Files (P0 — from RELEASE_READINESS_AUDIT)

The 2026-05-12 re-audit identified unresolved Git merge conflict markers in 9 Python files. These block parsing and may cause additional CI failures.

Files affected:
