**Issue:** Tests used `url.includes()` for URL matching which caused `/compliance/logs?job_id=job-1` to incorrectly match `/jobs/job-1` pattern.

**Attempted Fix:**
```typescript

// AFTER: Proper pathname extraction with URL helper
const getPathname = (url: string): string => {
  try {
    return new URL(url, 'http://localhost').pathname;
  } catch {
    return url;
  }
};
if (pathname === '/jobs/job-1') { ... }
```

**Result:** URL matching issue resolved, but deeper async timing issues with React Query state management remain. Tests remain skipped with updated documentation.

**Impact:** Documented root cause for future developers attempting to enable these tests.
