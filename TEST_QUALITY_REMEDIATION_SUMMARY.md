## Root Cause Analysis

### 1. AuthContext.test.tsx (5 errors + 3 failed tests)
**Issues Found:**
- `isLoading` state not set during `initiateLogin()` - implementation missing loading state management
- `act().rejects.toThrow()` pattern doesn't work with React Testing Library
- MSW (Mock Service Worker) interference with manual `fetch` mocks
- State mismatch in callback tests (sessionStorage vs callback params)

**Fixes Applied:**
- Added `setIsLoading(true/false)` to `initiateLogin()` in AuthContext.tsx
- Refactored error tests to use proper async error handling with try/catch
- Replaced manual fetch mocks with MSW `server.use()` handlers
- Fixed state parameter mismatch in callback tests

**Files Modified:**
- `frontend/client/src/contexts/AuthContext.tsx`
- `frontend/client/src/contexts/AuthContext.test.tsx`

---

### 2. useAuth.test.ts (6 failed tests)
**Issues Found:**
- `vi.mock()` called inside `beforeEach` - vitest mocks are hoisted, so this pattern doesn't work
- `mockNavigate` was scoped per describe block instead of module level

**Fixes Applied:**
- Moved `vi.mock('wouter', ...)` to top-level module scope
- Moved `mockNavigate` declaration to top-level with proper reset in `beforeEach`
- Fixed mock cleanup pattern

**Files Modified:**
- `frontend/client/src/hooks/useAuth.test.ts`

**Fixes Applied:**

## Test Quality Improvements

### Patterns Established
1. **Use MSW for API mocking** - Don't mock `global.fetch` directly when MSW is active
2. **Top-level vi.mock()** - All vitest mocks must be at module level, not in lifecycle hooks
3. **beforeEach for MSW handlers** - Register MSW overrides in beforeEach to survive `resetHandlers()`
4. **Async error handling** - Use try/catch in components, not `act().rejects` pattern

### Code Quality Fixes
- Added missing loading state management in AuthContext
- Fixed API endpoint drift between implementation and tests
- Proper cleanup and isolation between tests
